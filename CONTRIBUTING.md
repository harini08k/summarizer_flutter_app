# Contributing Guidelines

Thank you for considering contributing to the Document Summarizer App!  
All kinds of contributions are welcome: bug fixes, improvements, or new features.

## How to Contribute

1. **Fork the repository**  
2. **Clone your fork**  
```bash
git clone https://github.com/your-username/repository-name.git
cd repository-name
````

3. **Create a branch for your feature/fix**

```bash
git checkout -b feature-name
```

4. **Make your changes**
5. **Commit your changes with a clear message**

```bash
git commit -m "feat: add short summary feature"
```

6. **Push your branch**

```bash
git push origin feature-name
```

7. **Open a Pull Request** and explain your changes.

## Code Guidelines

* Write clear, readable, and well-documented code.
* Follow Flutter/Dart formatting for frontend and PEP8 for Python backend.
* Test your changes before committing.

## Reporting Issues

If you find a bug or want to suggest a feature, open an issue with:

* Clear title
* Steps to reproduce (for bugs)
* Expected vs. actual behavior
* Screenshots/logs if applicable

Thanks for helping make this project better!
